# VLA架构调研报告

**日期**: 2026-05-17  
**目标**: 评估VLA（Vision-Language-Action）架构应用于《黑神话：悟空》AI的可行性

---

## 一、核心结论

**VLA架构比纯行为克隆更适合游戏AI**，但需要大量算力和数据。

| 对比维度 | 行为克隆（BC） | VLA架构 |
|---------|---------------|---------|
| 问题建模 | 像素→动作（黑盒映射） | 像素+语言指令→动作（有意图理解） |
| 泛化能力 | 弱（过拟合训练场景） | 强（语言模型预训练提供先验） |
| 可解释性 | 无 | 有（可以问"为什么这样做"） |
| 数据需求 | 中（几千帧可训练） | 高（需要大量多样化数据） |
| 硬件需求 | 低（RTX 2060可训练） | 高（需要大显存GPU） |
| 训练成本 | 低（几小时） | 高（几天到几周） |

## 二、OpenDriveVLA架构分析

**论文**: [OpenDriveVLA: Towards End-to-end Autonomous Driving](https://arxiv.org/abs/2503.23463)  
**机构**: 慕尼黑工业大学  
**会议**: AAAI 2026  
**GitHub**: https://github.com/DriveVLA/OpenDriveVLA

### 架构设计

```
┌────────────────────────────────────────────────────────────┐
│  1. 3D Perception（感知模块）                              │
│     ├─ Backbone: ResNet50 → 2D视觉特征                     │
│     └─ BEV Encoder: 2D→鸟瞰图特征（统一空间表示）          │
├────────────────────────────────────────────────────────────┤
│  2. Env Token Injection（环境令牌注入）                   │
│     ├─ EnvQFormer: 提取智能体(Agent)令牌                   │
│     ├─ MapQFormer: 提取地图(Map)令牌                      │
│     └─ SceneSampler: 场景令牌采样（控制序列长度）          │
├────────────────────────────────────────────────────────────┤
│  3. Language Model（语言模型）                             │
│     ├─ 基座: LLaVA-style VLM                               │
│     └─ 输入: 视觉令牌 + 文本指令（"左转"、"绕开障碍"）    │
├────────────────────────────────────────────────────────────┤
│  4. Action Head（动作头）                                  │
│     ├─ 轨迹预测: 未来5秒路径点                             │
│     └─ 控制输出: 转向角、油门、刹车                        │
└────────────────────────────────────────────────────────────┘
```

### 关键创新

1. **BEV表示**：将多视角摄像头统一到鸟瞰图，消除遮挡和透视畸变
2. **令牌化**：将视觉特征转为语言模型可处理的Token序列
3. **文本指令**：可用自然语言指定高层目标（"左转"、"避让行人"）

## 三、OpenVLA架构分析（机器人）

**论文**: [OpenVLA: An Open-Source Vision-Language-Action Model](https://arxiv.org/abs/2406.09246)  
**机构**: MIT, Stanford, UC Berkeley, CMU, Google DeepMind  
**状态**: 开源（2024年8月发布）

### 模型规格

| 参数 | OpenVLA | RT-2（Google） |
|------|---------|----------------|
| 参数量 | **7B** | 55B |
| 训练数据 | 97万机器人轨迹 | 35万轨迹 |
| 视觉编码器 | SigLIP + DinoV2（双通道） | ViT（单通道） |
| 基座LLM | Llama 2 7B | PaLM-E |
| 开源状态 | ✅ 完全开源 | ❌ 闭源 |

### 关键技术

1. **双视觉编码器**：
   - SigLIP：语义理解（400亿图文对训练）
   - DinoV2：空间几何（12亿纯图像训练）
   - 组合后空间对齐任务成功率提升23%

2. **数据清洗**：
   - 移除12%无效轨迹（全零动作）
   - 平衡不同机器人平台数据
   - 训练效率提升40%

3. **微调方案**：
   - LoRA微调：仅97M可训练参数
   - 全量微调：197M可训练参数
   - 单张消费级GPU 8小时完成训练

## 四、硬件需求评估

### OpenDriveVLA（自动驾驶）

| 组件 | 最低配置 | 推荐配置 |
|------|---------|---------|
| GPU | RTX 3090 24GB | A100 40GB+ |
| CPU | Intel i7 12代 | Xeon 8核+ |
| 内存 | 64GB | 128GB+ |
| 存储 | 500GB SSD | 2TB NVMe |
| 训练时间 | - | 数周到数月 |

**注**: OpenDriveVLA官方尚未发布完整训练代码，以上为基于论文推测。

### OpenVLA（机器人）

| 任务 | GPU需求 | 显存占用 | 推理速度 |
|------|---------|---------|---------|
| 推理（FP16） | RTX 3060 12GB | ~14GB | 10-20 Hz |
| 推理（量化4-bit） | RTX 2060 6GB | ~4GB | 5-10 Hz |
| LoRA微调 | RTX 3090 24GB | ~20GB | 8小时 |
| 全量微调 | A100 40GB | ~35GB | 数天 |

### 《黑神话：悟空》适配评估

**用户硬件**: RTX 2060 6GB

| 方案 | 可行性 | 限制 |
|------|--------|------|
| 直接用OpenVLA推理 | ❌ 不可行 | 显存不足（需14GB） |
| OpenVLA量化4-bit | ⚠️ 勉强可行 | 推理慢，可能卡顿 |
| 自研轻量VLA（<1B） | ✅ 可行 | 需要自己设计架构 |
| 继续BC方案 | ✅ 最可行 | 6GB显存完全够用 |

### 📅 阶段二：轻量化VLA实现（1-2月）

**目标**: 实现适合游戏场景的轻量VLA

**架构设计**:
```
输入层:
  ├─ 视觉: 4帧游戏画面（224×224 RGB）
  └─ 语言: 文本指令（"前往boss"、"探索地图"）

编码层:
  ├─ 视觉编码器: ResNet18（轻量）或 MobileNetV3
  └─ 文本编码器: DistilBERT（轻量）

融合层:
  └─ Cross-Attention: 视觉特征与文本特征对齐

决策层:
  ├─ 动作分类: WASD（5类）+ 攻击/闪避（3类）
  └─ 鼠标回归: dx/dy（连续值）
```

**硬件适配**:
- 模型参数量: <500M（BC模型11.4M）
- 显存占用: <2GB（推理时）
- 推理速度: >10 FPS

**任务清单**:
- [ ] 搭建双编码器架构（视觉+文本）
- [ ] 实现Cross-Attention融合模块
- [ ] 收集带文本标注的训练数据（"正在前往X点"、"正在攻击Y敌人"）
- [ ] 训练轻量VLA模型
- [ ] 对比BC和VLA的泛化能力

**输出**:
- 开源代码仓库
- 训练好的轻量VLA模型
- 对比实验报告
