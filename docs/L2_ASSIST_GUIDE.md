# L2 辅助驾驶系统使用指南

**灵感来源**: 自动驾驶 L2 级别的"人机共驾"模式

---

## 核心理念

```
自动驾驶 L2：
  - 系统负责：加速/刹车、转向修正、紧急制动
  - 人类负责：全局决策、路线选择、复杂场景

游戏 AI L2：
  - AI 负责：自动闪避、自动面敌、自动连招
  - 玩家负责：战术决策、移动走位、技能释放
```

**关键洞察**：
> AI 负责局部、实时、高精度的反应式操作
> 玩家保留全局、战术性、创造性决策权

## 快速开始

### 1. 使用规则方法（无需训练）

```bash
# 直接运行（使用基于规则的自动闪避）
python assist/runner.py
```

规则方法原理：
- 检测画面运动幅度
- 运动超过阈值 → 触发闪避
- 无需训练数据，立即可用

### 2. 训练模型方法（更精准）

#### 步骤 1：采集训练数据

```bash
# 采集闪避数据（300秒，15fps）
python assist/data_collector.py --mode dodge --duration 300 --fps 15
```

**采集方法**：
1. 打开游戏，进入虎先锋战斗
2. 运行采集脚本
3. 专注于闪避操作：敌人攻击时按 Space
4. 其他时间保持不动或轻微移动
5. 采集 300-500 个闪避样本

#### 步骤 2：训练模型

```bash
# 训练闪避检测模型
python assist/train_dodge.py --data "l2_data/dodge_data_*.h5" --epochs 50
```

#### 步骤 3：使用训练好的模型

```python
from assist.runner import L2AssistRunner

# 加载训练好的模型
runner = L2AssistRunner(dodge_model_path="checkpoints/auto_dodge_best.pt")

# 游戏循环
while True:
    frame = capture.grab()
    action = runner.step(frame)
    executor.execute(action.action_id)
```

## 配置说明

在 `config.py` 中的 `L2_ASSIST` 配置：

```python
L2_ASSIST = {
    "enabled": True,  # 全局开关

    # 自动闪避（AEB）
    "auto_dodge": {
        "enabled": True,
        "confidence_threshold": 0.85,  # 置信度阈值（越高越保守）
        "cooldown_ms": 500,            # 闪避冷却时间
    },

    # 自动面敌（LCC）
    "auto_face": {
        "enabled": True,
        "deadzone_px": 30,             # 死区（像素）
        "sensitivity": 0.5,            # 鼠标灵敏度
        "smoothing": 0.3,              # 平滑系数
    },

    # 仲裁器
    "arbitrator": {
        "player_priority": True,       # 玩家输入优先
        "ai_override_threshold": 0.95, # AI 覆盖阈值
    },
}
```

## 人机协同逻辑

```
玩家按了攻击键？
  ├─ 是 → 执行玩家攻击
  └─ 否 → AI 检测到需要闪避？
              ├─ 是且置信度>95% → AI 自动闪避（覆盖玩家）
              └─ 否 → 玩家控制 / AI 面敌修正
```

**核心原则**：
1. **玩家优先**：玩家按了键，AI 不覆盖（除非紧急情况）
2. **高置信度覆盖**：AI 置信度 > 95% 时可以覆盖玩家
3. **平滑协同**：鼠标修正平滑执行，不干扰玩家操作

## 使用场景

### 场景 1：Boss 战（虎先锋）

```
玩家：控制移动、选择攻击时机
AI：
  - 自动闪避虎先锋的攻击（AEB）
  - 微调朝向始终面向虎先锋（LCC）
  - 玩家专注于战术决策
```

### 场景 2：探索跑图

```
玩家：控制方向、选择路线
AI：
  - 关闭自动闪避（无敌人）
  - 自动面敌关闭（无目标）
  - 完全由玩家控制
```

### 场景 3：小怪清理

```
玩家：选择攻击目标
AI：
  - 自动闪避小怪攻击
  - 自动面敌最近的敌人
  - 玩家专注于选择目标
```

## 与完全自主 AI 的对比

| 方面 | 完全自主 AI | L2 辅助驾驶 |
|------|------------|-------------|
| **数据需求** | 海量（所有场景） | 少量（特定功能） |
| **训练时间** | 数天到数周 | 数小时 |
| **模型复杂度** | 高（通用模型） | 低（专用模型） |
| **可控性** | 低（黑盒） | 高（可调参数） |
| **用户体验** | 替代玩家 | 辅助玩家 |
| **开发周期** | 长 | 短 |
| **风险** | 高 | 低 |

## 常见问题

### Q: AI 会不会干扰我的操作？

A: 不会。系统设计为"玩家优先"，只有在高置信度且玩家无输入时才介入。

### Q: 如何调整 AI 的介入程度？

A: 修改 `config.py` 中的 `confidence_threshold`，值越高越保守。

### Q: 规则方法和模型方法哪个更好？

A:
- **规则方法**：简单、无需训练、但准确率低
- **模型方法**：准确率高、但需要训练数据
- 建议先用规则方法验证，再用模型方法提升

### Q: 如何采集高质量的训练数据？

A:
1. 专注于单一操作（如只闪避）
2. 保持其他操作最小化
3. 采集 300-500 个样本
4. 确保正负样本平衡
